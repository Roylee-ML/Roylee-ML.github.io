程序员读《格鲁夫给经理人的第一课》

Posted by Roylee on 2017-02-12

引子

这本书是前几日同事推荐的，是一本写给中层管理者的书。同时，也比较适合转向管理的程序员（而作者本人之前就是一名工程师）。

虽然自己仍然是个 Coder，但还是找来这本书读了一遍。其实，作为一个程序员，不应该每天只是埋头在 Code 的海洋中，也要注重自己的人生规划与管理。而我们也不仅仅是代码的搬运工，我相信，一个好的程序员一定也是一个好的产品经理——不想当管理者的程序员一定不是一个好设计。而作为管理者，第一件事当然也是如何管理自己。在这个意义上来看，我们每个人都有必要学习一些管理的知识。

作者

个人认为，读一本书首先要看看作者是谁。如果作者是个了不起的人物，他写的书正常来说也是比较好的书。而本书的作者。。。。。。。（TODO：介绍作者）

一些感触

这里只写些自己读完之后的一些理解与想法，对于书中的内容就不做过多的阐述。其实，在唐巧的文章写给程序员的管理入门课程 -《格鲁夫给经理人的第一课》里，已经对这本书做了精要的归纳。

怎样搞生产？

你必须按预定的时间、可接受的品质以及可能的最低成本，依据顾客的需求制造及运送产品。

而这里涉及的问题有：

  • 找出整个流程中的“限制步骤”
  • 依据“限制步骤”找出最佳策略
  • 最后测试与验货

首先，找出“限制步骤”是一项查看事务优先级的策略。所谓“限制步骤”是那些最昂贵、最困难、最敏感的部分。这样，我们在执行生产的过程中，会依据其中最难搞定的部分，来制定整个生产策略。

虽然，格鲁夫在第一部分用“早餐店的生产线”的例子阐述了这些问题，并指明对于一个早餐店来说，其限制步骤会是人员数量、设备产能、库存数量（格鲁夫只是给我们指明了方向）。但是，对于我们来说，恰恰这个找出“限制步骤”的过程才尤为重要。

那么，我们要如何做才能找出所做事项中的“限制步骤”呢？

从格鲁夫的经验来看，不难发现。重要的一点是要掌握足够充分的信息（而作者在之后的部分也列出了自己一天工作的内容，其中大部分的内容也是在收集信息），对于自己的资源以及所做之事的清楚认识。这也正是一个发现问题、分析问题的过程，而做到这点就要对问题有更清楚的认识，抛弃个人情绪更全面的了解事情的状况。而作为一个程序员，我们在实际工作中的限制步骤就可能是我们的专业技能、开发周期、功能需求等。

找到“限制步骤”之后，就要找出最佳的方案策略了。

格鲁夫认为，凡是能以最低成本达到理想的运送速度以及品质的，便是最佳方案（当然还是以早餐生产线为例）。简言之就是要以最低成本实现理想目标的方案就是最佳方案。而这里，我认为格鲁夫所指的成本不仅仅是物质上的成本，更应该是广义上的机会成本。那么，如何才能敲定一个最佳方案呢？

首先，要有方案。作者并没有阐述如何制定方案的过程，而仅仅是分析如何在各个方案中找出最佳的那个。那么，制定方案，就要充分发挥各个经理人的作用，在后文中，团队

2.效率应该是本书的重点；效率、创新、自强

3.一对一会议，会议的目的也要越来越单纯，否则失去了会议的作用

4.大浪淘沙，适者生存。不要不断的提高自己的技能

5.只有在变化中生存的人才会对未来抱持乐观的态度

6.code 不能只是应对产品的各种突发需求，否则造成的就是自己的混乱。（早餐店的生产流程）

早餐的生产线

1.找出限制步骤：最昂贵、最困难、最敏感的项目。但是，做到这一点也是至关重要的，这是一个发现问题、分析问题的过程，而这个能力却是十分重要的。

而作为程序员，在产品需求出来之后，关键的第一步就是完整的分析整个需求，是整个需求，同时在需求讨论中要充分的发挥自己的作用，已使自己更好的了解需求。

2.最低成本实现目的的方案就是最佳方案。但这里的成本，我认为并不是简单的物质成本，而是机会成本

3.及早发现、及早解决。这是在众多管理大师都会提到的一点。

4.找出限制步骤的前提是，要知道并清楚自己的资源。

5.指标可以认为是目标，是大目标下的具体小目标，同时有时可以衡量可以评估的。

对于指标的评估比较，每天要做总结，归纳自己的完成度。

6.重复印证表就像一个承诺，时刻的督促管理者的践行成果

团队战

1.杠杆率，工作简单化

着眼于产出是提高产能的关键，以结果为导向

这类似于 KPI，其实 KPI 并没有错，错的只是 KPI 的制定规则。就像一产出、指标为导向，而产出、指标的制定就极为重要，因为一旦有错误的指标以及产出标准，就会降低生产效率。在这一点上，同样可以采用预测与重复印证的方式，这样可以更加实时的看看到结果并做出及时的调整

2.收集信息的重要性。开篇的日本大圆桌与电子邮件也是说明信息的重要性，信息沟通增加彼此信息收集的概率

杠杆率提高效率第一步：收集更多的信息。“对大多数的经理人而言，最重要的信息往往来自于简短而非正式的谈话”（而这些谈话，又决定于一个团队氛围）—信息的价值，跟失效有绝对的关联

3.给予提示（建设性的建议与引导），重要于指定决策

4.会议只是没接，是为了传递信息、指定决策。关键的在于：必须找到最有效的没接已完成任务

5.在众多活动中找到重点

6.授权的必要条件：两者之间必须要有相同的信息基础，以及在开展工作和解决问题上有一套彼此认同的方法。

7.NOTE 经理人的概念：其影响范围也是一部分，所以，对于下属，相对的你的上级也是你的下属。

8.格鲁夫的开会概念更像是一个互通有无的过程。所以上司在扮演一个协调者、学生与教练于一身

9.一对一会议，可以认为是一对一交谈。管理的重要概念也是，不要当众戳穿一个问题，一对一解决是最好的

10.会议的价值，必要性，是否值得开？

11.决策，要充分发挥最了解问题的那些人的价值

12.会议的前提是说真话，发现真相，为此要建立“犯错难免”的氛围与文化

推动组织的巧手

1.集权放权：参与其中，合理建议，不过分干涉

2.双重报告，双重职责身份。充分了解信息，沟通。更灵活的组织结构，每个人员发挥自己的作用，但是有不至于其他人员脱节。（比如，一个领导理应向上级报告，但是同时在另一个组织中，这样他在另一个组织中的作用有很明显，不至于他的作用被局限化）

临时小组的概念，解决问题成立小组。完成后解散小组

谋事在人

1.经理人的最主要的职责就是激发下属的最佳表现。不同时代的管理风格也大多围绕着激励机制
